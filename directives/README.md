# Directives (Layer 1 — What to Do)

This folder contains **Markdown SOPs** — natural-language instructions that define objectives, inputs, tools/scripts to use, outputs, and edge cases.

## Guidelines

- Write directives as you would for a mid-level employee.
- Each directive should be self-contained and reference execution scripts by path.
- Update directives when you discover API constraints, better approaches, common errors, or timing expectations.
- **Never overwrite** a directive without asking unless explicitly instructed.

## Template

Use `_template.md` as a starting point for new directives.
